Learn XO keyboard, and the Terminal Activity

To set up for this lesson, please load the HelloWorld.activity from the examples window of the Project Page. Then click on the "Activity" tab, and exit into the Terminal "<ctl> D" .

Essential navigation shortcuts:

  1. <up arrow-pg up> == repeat previous commands. Repeated <up arrow>s gets you to earlier and earlier in the history.
  2. <fn><shift><up arrow> (3 keys at once) == scroll up one screen
  3. <ctl><alt><erase> (3 keys at once) == restart the Sugar desktop (use when nothing is working--all data will be lost).
  4. <frame> (upper right-most key) == switch between Sugar's currently available windows. Push again to hide frame.

Type "history 10", or to use tab completion, "hi <tab> 10". This shows the previous commands that you can get to with the up and down arrows on the keyboard. Issue the history command, without the number 10. This will include the full history since PyDebug was loaded on your machine, and could be very long. If longer than one page, use the <fn><shift><up arrow> to view older history. If there is a command you want to use again, just type "!<number> to reissue the command.

Using a pager

Type "cat pydebug.py" and hit <enter>. The "cat" command lists any file to the terminal screen. The "pydebug.py" file is the main file for the PyDebug activity. You can use the <fn><shift><up arrow> to see the earlier parts of the program. But it's not really convenient. The "less" program is designed to make it easier to look at long files. Type "less pydebug.py". Note that it stops after the first page. Use <fn><down arrow> and <fn><up arrow> to scroll up and down in the text. If it is easier, the "f" and "b" key allow you to do the same with a single keystroke. When you are done looking at the file, the "q" key is used to "quit". IPython uses the "less" pager most of the time when it is presenting long documents to you. One of the times is when you issue "?" at the IPython prompt. Go into IPython now (type "go"), and type "?". Check out the action of the "f","b",<fn><up arrow>,<fn><down arrow>, and "q" keys. Note that the pager doesn't wipe out what was on the screen before you started using it.

Learn about the second Terminal page

Click on the "Activity" tab of PyDebug, and then on the "HelloWorld.activity" tab at the bottom right of the sceen. When you get to this page, it will appear slightly whiter that the IPython screen (which is slightly yellow). Let's explore the navigation aids available on this terminal screen. First type "alias". The shell will respond with all the aliases that are currently defined for you. We will use the following:

  • pp == Go to the activity which is currently loaded into the PyDebug playpen (the program being debugged).
  • act == Go to the folder which holds the Activities that are loaded into to XO.
  • cd == Go to the home directory where PyDebug is permitted to write permanent files
  • olcp == Go to the main home directory for the 'olpc' user, where user data is stored.

The "pwd" will print the entire path for the working directory, and can be used to see the results of each of the above commands.

The other aliases on the list are helpful in making frequently used command more humanly readible or require fewer keystrokes and will be discussed in more detail in lesson 2
