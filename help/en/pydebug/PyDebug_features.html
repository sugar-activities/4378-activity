features of the debugger

  • integrated ipython console, editor, journal, home directory interface, and html browser based help system

  • breakpoint, single stepping,examination of variables and post mortem analysis after unhandled exception termination.
  • automatic XO bundle, manifest generation
  • simplified svg icon generator
  • sandbox execution of debugged activity in a separate process
  • synchronization of uncaught exceptions and errors with the integrated tabbed multi-file editor
  • collection of template models and examples for rapid activity development
  • dated snapshots of your activity to secure digital flash card or USB stick
  • a significant amount of documentation including James Simmons Writing Sugar Activities, python, pygtk tutorials available offline


quick tips

The second tab at the bottom right of the activities (ipython console screen) can be used to get to a terminal window. most of the functions of terminal (except for superuser --su) are available. Some convenience aliases have been defined:

  • pp - change directory to playpen

  • pdb -- cd to storage shelf for activities in process (On the project page, the full activity tree is copied from the playpen to the shelf when '~/pydebug/' is displayed above the middle window, and the left arrow is clicked.
  • cd -- will take you to the root of the disk space that PyDebug uses for its permanent storage.
  • du1 - list of the amount of disk used (can be used to find out how big your Activity will be when it is installed, and how big each of the first level directories is)
