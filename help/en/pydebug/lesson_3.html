Use IPython tab completion, and introspection

The "os" module in python permits applications to run on windows, linux, and Apple mac machines. It will be found in most of the Activites on the XO, and it has many functions. Suppose you want to use a function that is an Operating System function, but you don't know the name of the function. Introspection is your tool to find out.

Click on the "Activity" tab, type "import os". You can get a list of all the functions in "os" by typeing "os.<tab>" (The "." is required). The list is too long to fit on one screen, so you'll have to page up to see the whole list . Let's suppose that you want to find out how to trim off everything except the last part of a filename. As you look through the list, you might notice that there is an entry labeled 'path'.

So check it out. Type "os.path.<tab>". You get another long list. You see a few that seem promising (basename, curdir, dirname, normpath, pardir, relpath, splt, stat). The list is alphabetic, and so for fun you decide to start from the bottom.

Type "os.path.stat??". The ?? says you want as much information as possible. You get a lot of information. The <fn><shift><up arrow> doesn't seem to work. I tried hitting a lot of keys, and accidently hit the "h" key. I got to a help page for the "less" pager, which is what IPython uses whenever it has a lot of data to send to the screen. There is lots of information about which keys to use to control the pager, and it says you can use "q" to quit the help, which brings you back to the long listing about os.path.stat. Hitting "q" from there returns you to the IPython prompt.

So next try "os.path.split??". Did you <up arrow>, erase stat, and type "split" ? It looks like "split" could be used for your purposes. You can just throw away the tail, and the head is what you are looking for. But some of the other methods may accomplish what you wand also.

Skipping ahead, you will find that "os.path.basename??" describes exactly what you were looking for. So let's test it out. Type "os.path.ba<tab> completes to "in (20) os.path.basename". Now you are going to give it a long path, and see if it returns just the last part. So continue typeing "(/h"<tab> which is completed to "In (21) os.path.basename('/home/" . Then you type "o"<tab> --> os.path.basename('/home/olpc/. You type "A"<tab> which becomes os.path.basename('/home/olpc/Activities/. Then you type "Py"<tab>. It responds os.path.bassename(/home/olpc/Activities/PyDebug.activity. You type "py"<tab>. You notice the py also would select pytoolbar.py so you just finish it off, typing "debug.py')" and it responds with the answer you were execting .

Then you wonder if you could pick off the last portion of the path with the same function. You <up arrow> the last command, and backspace to erase the "pydebug.py". The result is not what you expected -- just an empty string "". If you delete the trailing "/", you get results more like you expected -- "PyDebug.activity".

PyDoc as an introspection tool

Go to the "HOME" of the Help menu (the round hollow circle) on the menu. Select PyDoc link and scroll down to find the /usr/lib/python section. Click on "os" which will take you to information that is available from PyDoc on the OS module.

You will find "basename", "dirname", and "split" listed under functions, but not nearly as much information about the source, inputs, and return values as was available from IPython.

Dir() as an introspection tool

Return to the Activity tab, and type "dir(os)". Following this type "dir(os.path)". This demonstrates the native ability python to document itself. The "dir" command is the raw lookup tool that comes with the standard python command line interpreter. I find the IPython interface, with its tab completion, and the ability to narrow the possibilities by typing a few characters, to be much more useful.
